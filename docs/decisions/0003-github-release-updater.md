# 0003：采用 GitHub Release 自定义更新流程

- 状态：已接受
- 日期：2026-07-19

## 背景

QuickPaste 是个人维护的 Windows 项目。维护者明确决定不使用 Tauri updater 的更新签名，也暂不购买 Windows Authenticode 证书；更新体验参考 VoxType，通过 GitHub Release 检查新版本、下载 NSIS 安装包并启动更新。

## 决策

- 客户端只访问固定的公开仓库 `zkwi/QuickPaste`，不允许用户改写更新源，也不在客户端保存 GitHub Token。
- 启动后只做后台版本检查，不自动下载或安装；后台无更新和网络失败不打扰用户。用户可以在设置页和托盘手动检查。
- 预览阶段读取已发布且非草稿的 GitHub Releases，包括 Pre-release，并使用 SemVer 选择高于当前版本的最新版本。
- 只接受名称与目标版本完全匹配的 `QuickPaste_<version>_x64-setup.exe`，拒绝 MSI、portable、其他架构和模糊匹配的可执行文件。
- 下载使用 HTTPS、有限超时、128 MiB 大小上限和唯一临时文件。自动安装强制要求 GitHub `sha256:` 摘要；缺失或不一致时禁用或中止自动安装。
- 下载完成且摘要正确后，前端再次刷新待写入的剪贴板历史；刷新失败时禁止启动安装器，避免退出造成数据丢失。
- 启动 NSIS 前重新读取文件并核对大小和 SHA-256；启动失败或立即非零退出时保持当前进程运行。安装器成功启动后通过既有受控退出链再次刷新历史，再退出当前版本。
- 自动检查默认开启，每个进程延迟执行，并在本机缓存检查时间以避免 24 小时内重复请求。请求只包含 GitHub 所需的网络元数据和 `QuickPaste/<version>` User-Agent，不上传剪贴板、设置或设备标识。

## 用户体验

- 设置页显示当前版本、检查状态、最新版本、安装包大小、下载进度和明确的失败信息。
- 发现更新时使用非抢焦点提示；只有用户点击“下载并安装”后才开始下载。
- 托盘“检查更新”打开设置页并复用同一前端状态机，不在 Rust 与 Vue 中维护两套更新状态。
- 中英文均覆盖未检查、检查中、已最新、发现更新、下载中、校验失败与安装启动失败等状态。

## 安全边界

SHA-256 只用于发现下载损坏或资产不一致，不等同于发布者签名。GitHub 账户或 Release 被攻破时，此方案不能提供独立的密码学发布者验证；这是维护者在个人项目阶段明确接受的风险。未来若改用 Tauri updater 或 Authenticode，必须新增迁移 ADR，不能在不兼容的情况下静默替换更新信任根。

`spawn` 成功仅表示 Windows 已接受安装器进程，不等于整个安装最终成功。QuickPaste 会识别安装器的立即非零退出，但安装器独立运行后的失败无法由旧版本可靠观察；发布说明和界面不得承诺“安装失败一定不退出”。

## 验证

- 纯函数测试覆盖 SemVer、Release 选择、严格资产名、大小限制、摘要解析和安装参数。
- 平台适配测试覆盖非 Tauri 降级、并发保护、进度、失败和迟到结果。
- UI 测试覆盖自动检查静默行为、手动检查、设置迁移、历史刷新失败和中英文状态。
- 发布前人工覆盖断网、404、畸形 JSON、缺失/错误摘要、下载中断、安装启动失败和 `0.1.0 → 0.1.1` 更新。
