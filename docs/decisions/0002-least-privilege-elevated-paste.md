# 0002：管理员窗口使用最小权限的一次性粘贴 helper

- 状态：Accepted
- 日期：2026-07-19

## 背景

普通权限进程不能可靠地向高完整性窗口发送输入。让整个剪贴板应用长期以管理员权限运行，会扩大剪贴板历史、UI 和第三方内容处理的权限边界。

## 决策

QuickPaste 主进程始终以普通用户权限运行；若检测到管理员权限则提示并退出。只有用户明确向已复核的管理员目标粘贴时，才启动带时限、单用途授权的一次性 helper。

主进程在 UAC 前创建随机命名的本地管道并保有服务端句柄；命令行只传父进程 PID、绝对截止时间和随机会话标识，不传目标窗口。连接建立后，主进程必须确认客户端 PID 等于本次 `ShellExecuteExW` 返回的 helper 进程，helper 必须确认服务端 PID 等于声明的父进程、父进程未提权且双方运行同一可执行文件。helper 使用 Identification 级 SQOS 连接，避免恶意服务端借管道冒用其高权限令牌。只有互认成功后目标请求才通过管道传输，且不会写入磁盘。

写入待粘贴内容后，主进程必须读回验证其内容，并在剪贴板序列号前后保持一致时将该序列绑定到请求。helper 在收到请求、聚焦目标以及发送输入前重复核对序列；任何不一致都降级为仅复制，避免 UAC 等待期间被替换的内容进入管理员窗口。

同一时刻只允许一个 UAC helper 请求。界面等待超时后设置取消标记；即使系统授权提示稍后才返回，迟到的 helper 也会被终止且不会收到目标请求，新的请求在旧流程真正结束前失败关闭。

## 结果

- UAC 只在确有需要时出现，主历史数据库和常驻进程不获得管理员权限。
- UAC 取消、超时、目标变化或焦点失败必须降级为仅复制，不能重试到未知窗口。
- helper 参数、会话标识和目标不得写入日志；目标、凭据和签名材料不得写入仓库或授权文件。
- 每次修改授权契约都必须有 Rust 单元测试和真实管理员窗口验收。

## 重新评估条件

仅当 Windows 提供更小权限且可验证的官方输入机制，或产品明确取消管理员窗口自动粘贴时，才替换该方案。
