# 0004：数据库跟随 QuickPaste 可执行文件

- 状态：已接受
- 日期：2026-07-20

## 背景

QuickPaste 同时提供当前用户 NSIS 安装包和绿色版 ZIP。开发阶段使用 Tauri 应用数据目录会让数据库与程序分离，用户难以直接复制完整历史，也容易误以为绿色版是完全便携的。

## 决策

- 正常运行时，数据库固定写入 `QuickPaste.exe` 同级的 `data` 文件夹；SQLite 的 WAL、SHM、恢复隔离文件也留在这一目录树中。
- 安装版和绿色版使用完全相同的路径规则，不通过注册表、环境变量或版本号分叉。
- 迁移必须在完全退出 QuickPaste 后复制整个 `data` 文件夹，不能只复制正在使用的 `history.sqlite3`。
- 设置页只展示稳定的相对位置说明，并通过窄 IPC 让 Windows 资源管理器打开真实目录；前端不接收用户绝对路径。
- 开发阶段旧应用数据目录不读取、不扫描、不自动迁移。验收模式继续要求显式隔离 profile，避免测试写入真实程序目录。

## 生命周期与限制

当前用户 NSIS 安装目录位于用户可写位置。Tauri NSIS 卸载器只删除安装清单中的程序与资源，并以非递归方式移除安装目录，因此包含 `data` 时目录会保留；覆盖安装同样不会改写该文件夹。绿色版应解压到用户可写目录，不能直接在只读介质中运行。

数据库与导出备份仍未提供应用层加密。便于复制也意味着任何能读取该目录的进程都可能读取历史；用户应结合 Windows 账户权限和磁盘加密保护数据。

## 验证

- Rust 单元测试固定验证 `QuickPaste.exe → data` 映射及验收 profile 优先级。
- 前端平台测试验证打开目录命令不返回路径，组件和应用测试验证中英文提示、按钮事件及失败反馈。
- 发布候选继续执行 `npm run check`、Windows x64 NSIS 构建，并分别核验安装包和绿色版 ZIP。
