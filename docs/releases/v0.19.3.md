# QuickPaste v0.19.3 发布说明

v0.19.3 修复从快速面板返回 Chrome 等多线程窗口时，Windows 前台输入焦点交接可能失败并退化为仅复制的问题。本版本以标准 GitHub Release 分发，不标记为 Pre-release；真实机长循环仍保留为待验证项，因此不表述为已经全面验收。

## 主要变化

- 自动粘贴在切换回原目标前，临时与当前前台窗口线程共享 Windows 输入队列，使后台 Tauri 命令线程可以可靠完成前台焦点交接。
- 目标顶层窗口成为前台后保留 100ms 稳定期，随后再次核对目标进程、实际输入焦点子窗口、剪贴板版本和授权时限，再发送 `Ctrl+V`。
- 目标失效、剪贴板被其他应用改写、修饰键未释放或完整性级别不匹配时，仍只复制并显示手动粘贴提示，不会向未经确认的窗口注入输入。
- 实现路径参考 [Ditto `d36f864` 的 `ExternalWindowTracker`](https://github.com/sabrogden/Ditto/blob/d36f864f9e6bc3558e11e3f1c9f5f522b8079702/src/ExternalWindowTracker.cpp)，但不修改系统全局前台锁超时，也不强制释放用户正在按下的全部按键。

## 数据与兼容性

- 本版本不改变 SQLite schema、数据目录、历史保留规则、剪贴板格式或管理员授权协议，无需迁移。
- 数据库、WAL 和 SHM 继续保存在 `QuickPaste.exe` 同级的 `data` 文件夹。复制或迁移前必须从托盘完全退出 QuickPaste，避免遗漏仍在 WAL 中的数据。
- 主程序继续保持普通用户权限；管理员窗口仍只通过现有一次性 UAC helper 粘贴。
- 安装包仍只提供当前用户范围的 Windows x64 NSIS，不生成 MSI、MSIX 或 APPX。
- 当前发布二进制未进行 Authenticode 签名。
