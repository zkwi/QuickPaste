# QuickPaste v0.17.1 发布说明

v0.17.1 修复 Chrome 富文本记录通过双击或 Enter 直接粘贴时可能无响应，并被错误记录为 QuickPaste 新条目的问题。本版本以标准 GitHub Release 分发，不标记为 Pre-release；真实机长循环仍保留为待验证项，因此不表述为已经全面验收。

## 主要变化

- 写入文本、HTML、RTF 或文件格式包后，优先执行完整格式回读校验；若外部读取器恰好短暂占用系统剪贴板，则使用写入后捕获的剪贴板序列继续恢复目标窗口并发送粘贴。
- QuickPaste 写回复杂剪贴对象时附加专用内部标记，监听器忽略自身写入，不再生成 QuickPaste 来源的重复历史记录。
- 内部标记不替换或裁剪标准剪贴板格式；Chrome 仍可接收 HTML 超链接与多段富文本，记事本等纯文本目标仍可选择 Unicode 文本表示。

## 数据与兼容性

- 本版本不改变 SQLite schema、数据目录或历史保留规则，无需迁移。
- 数据库、WAL 和 SHM 继续保存在 `QuickPaste.exe` 同级的 `data` 文件夹。复制或迁移前必须从托盘完全退出 QuickPaste，避免遗漏仍在 WAL 中的数据。
- 安装包仍只提供当前用户范围的 Windows x64 NSIS，不生成 MSI。

## 尚待真实机验证
