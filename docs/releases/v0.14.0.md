# QuickPaste v0.14.0 发布说明

v0.14.0 让安装版和绿色版使用同一套可复制数据目录，并加强首次引导、窗口拖动和存储管理体验。本版本以标准 [GitHub Release](https://github.com/zkwi/QuickPaste/releases/tag/v0.14.0) 分发，不标记为 Pre-release；真实机长循环仍保留为待验证项，因此不表述为已经全面验收。

## 重点变化

- 数据库、WAL 和 SHM 统一保存在 `QuickPaste.exe` 同级的 `data` 文件夹。完全退出 QuickPaste 后复制整个文件夹，即可在安装版或绿色版之间迁移历史。
- 设置页继续只展示数据库大小和数据条数两项核心指标，并新增简洁的数据位置说明和“打开数据目录”按钮。
- 首次引导由原生窗口状态托管：首次运行主动居中并保持显示，不会因失焦莫名隐藏；托盘、快捷键或重复启动应用时仍返回引导。
- 快速面板、管理页和首次引导的顶部非交互区域统一使用 Tauri 原生拖动区域，可像普通窗口标题栏一样按住移动。
- 完成首次引导后恢复日常逻辑：快捷键和托盘点击继续捕获当前粘贴目标，并优先在文本光标附近显示。

## 数据位置与兼容性

- `0.14.0` 不读取或迁移开发阶段旧版本的 Tauri 应用数据目录，会在新位置创建空数据库。
- 迁移前必须从托盘完全退出 QuickPaste；程序运行时只复制 `history.sqlite3` 可能遗漏 WAL 中尚未合并的数据。
- NSIS 覆盖安装不会改写 `data`；卸载后含有用户历史的非空程序目录会保留，确认不再需要备份后可手动删除。
- 轻量 UI 设置仍由 WebView 本地状态保存，不属于 SQLite 数据库复制范围。
- 数据库和导出备份没有 QuickPaste 应用层加密，请结合 Windows 账户权限和磁盘加密保护。
- 安装包仍只提供当前用户范围的 Windows x64 NSIS，不生成 MSI。
