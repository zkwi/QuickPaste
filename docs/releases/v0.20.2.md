# QuickPaste v0.20.2 发布说明

v0.20.2 修复长文本及包含换行的富文本记录在 Codex 等应用中更容易退化为仅复制的问题。本版本以标准 GitHub Release 分发，不标记为 Pre-release；真实机长期矩阵仍保留为待验证项，因此不表述为已经全面验收。

## 主要变化

- 本机内容无关诊断确认短文本通常只携带纯文本格式，而较长、多行的浏览器/Codex 记录更常同时携带 HTML，因此进入不同的格式包写入路径。
- 旧实现仍持有 Windows 剪贴板句柄时就读取序列号；随后关闭剪贴板产生最终稳定序列，粘贴前的安全校验会把这一变化误判为外部修改并停止发送 `Ctrl+V`。
- HTML、RTF 与文件格式全部写完后，QuickPaste 现在先显式关闭剪贴板，再读取并保存最终序列号；这一顺序也与 Ditto 完成剪贴板交接后再激活目标并发送粘贴的做法一致。
- 新增真实 Windows 剪贴板回归断言，要求写入函数返回值与关闭剪贴板后的稳定序列完全一致。

## 数据、安全与兼容性

- 本版本不改变 SQLite schema、历史数据目录、剪贴板载荷格式、设置 schema 或管理员授权协议，无需迁移。
- 目标窗口、焦点、剪贴板版本、物理修饰键、helper 授权时限和管理员权限安全校验保持不变；只修正格式包写入返回的序列号时点。
- 主程序继续保持普通用户权限；管理员窗口仍只通过现有一次性 UAC helper 粘贴。
- 安装包仍只提供当前用户范围的 Windows x64 NSIS，不生成 MSI、MSIX 或 APPX。
- 当前发布二进制未进行 Authenticode 签名。
