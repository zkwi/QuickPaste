# QuickPaste v0.13.0 发布说明

v0.13.0 优化首次启动、窗口拖动和托盘唤起体验。本版本以标准 [GitHub Release](https://github.com/zkwi/QuickPaste/releases/tag/v0.13.0) 分发，不标记为 Pre-release；真实机长循环仍保留为待验证项，因此不表述为已经全面验收。

## 重点变化

- 首次安装时，引导窗口会居中显示；完成引导后，日常快捷唤起仍继续跟随文本光标或鼠标。
- 引导最后一步可选择添加一条欢迎记录并练习快捷粘贴，也可以直接开始。欢迎记录只写入 QuickPaste 本地历史，不读取、替换或写入用户当时的系统剪贴板内容。
- 练习提示可随时关闭；再次通过快捷键或托盘打开快速面板时会优先选中欢迎记录，成功粘贴后自动结束提示。
- 无边框窗口顶部的非按钮区域支持按住鼠标拖动，窗口控制按钮仍保持独立操作。
- 左键点击托盘图标和托盘“打开”与全局快捷键共用同一条目标捕获、光标定位和显隐路径，不再清空待粘贴目标。

## 隐私与兼容性

- 欢迎练习记录属于应用本地历史，不会为了创建示例而调用系统剪贴板写入接口。
- 设置 schema 从 5 升级到 6，仅新增练习提示状态；SQLite 数据库格式没有变化，不需要迁移历史数据。
- OCR 仍只使用 Windows 本机能力，不下载模型、不上传图片或识别文字。
- 安装包仍只提供当前用户范围的 Windows x64 NSIS，不生成 MSI。

## 尚待真实机验证

- Windows 10/11 的覆盖安装、卸载与重装。
- 100%–250% DPI、多显示器负坐标和任务栏边界。
- 托盘点击后的跨应用光标定位、管理员目标回贴和 WebView2 缺失路径。
- 长时间剪贴板监听与首次引导完整安装生命周期。

以上项目保持为 `pending real-machine`。维护者选择将 v0.13.0 作为标准 GitHub Release 分发，但这不改变上述验证状态。
