# QuickPaste v0.19.3 验证报告

验证日期：2026-07-23

本报告记录 v0.19.3 Windows 前台焦点交接与自动粘贴修复的代码回归、自动化和候选产物证据。本次不使用 Computer Use；真实应用完整交互矩阵、真实设备长时间运行、完整安装生命周期和完整 DPI 矩阵继续单独列为 `pending real-machine`。

## 根因与缺陷回归

- 修复前，粘贴命令从 Tauri 后台线程直接调用 `BringWindowToTop` 和 `SetForegroundWindow`，只等待目标顶层 HWND 成为前台；调用线程没有与当前前台窗口线程共享输入队列，Chrome 等多线程窗口的子焦点交接失败时会安全退化为仅复制。
- 对照 Ditto `ExternalWindowTracker::ActivateTarget` 与 `SendPaste`：目标切换前应临时 `AttachThreadInput` 到当前前台线程，并在确认目标激活后留出稳定时间再发送粘贴键。
- 修复前，新增的焦点交接顺序测试在缺少 `attach_to_foreground_thread()` 时按预期失败；加入最小实现后，同一测试转绿。
- 修复后仅在前台切换阶段持有 RAII 输入队列附件，离开作用域即解除；目标稳定期结束后再次验证目标身份、焦点、剪贴板序列和 helper 时限。
- 未采用 Ditto 修改系统全局 `FOREGROUNDLOCKTIMEOUT` 和强制弹起全部按键的做法，避免影响其他应用与用户当前输入。

## 自动化与安全审计

- `npm ci`：通过，共安装 140 个锁定包。
- 治理脚本测试：46 项通过；版本同步检查已确认 QuickPaste 0.19.3。
- 前端测试：34 个测试文件、727 项测试通过；TypeScript 检查与 Vite 生产构建通过。
- Rust 格式检查与 Clippy（`-D warnings`）：通过。
- 焦点交接、目标生命周期与 `Ctrl+V` 输入序列定向测试：通过。
- `npm run check` 最终完整复核通过：Rust 格式、Clippy（`-D warnings`）及 321 项 Rust 测试全部通过；系统剪贴板往返测试在已安装 QuickPaste 正常退出后通过。
- `npm audit --omit=dev --audit-level=high`（官方 npm registry）：通过，0 个漏洞。
- `cargo audit --file src-tauri/Cargo.lock`：退出码为 0，没有阻断性漏洞；报告 17 条允许的维护状态或 soundness 警告，来自 Tauri 的 GTK3/UNIC 间接依赖。QuickPaste 当前只交付 Windows，这些依赖不进入本次 Windows 可执行文件；警告仍由每周 RustSec 工作流持续监控。
- `npm run build:windows`：通过，只生成 v0.19.3 x64 NSIS 候选安装包，未生成 MSI、MSIX 或 APPX。
