# QuickPaste v0.7.0 发布说明

v0.7.0 集中修复图片浏览、来源应用图标和设置页布局回归，并把已经默认启用的 Windows 本地 OCR 状态清晰呈现在列表与预览中。本版本以[未签名 GitHub Pre-release](https://github.com/zkwi/QuickPaste/releases/tag/v0.7.0)分发；真实机长循环仍保留为待验证项，因此不表述为稳定版已全面验收。

## 主要变化

- 图片缩略图恢复：原生历史摘要保留安全、受限的来源应用图标；图片列表通过独立 IPC 按需生成最长边 96 像素的 PNG 缩略图，不加载完整图片正文。
- 图片预览收紧：减少标题、格式和提示区域占用，让图片在首屏剩余空间内尽可能完整展示，不再因固定高度留下大块无效空间。
- OCR 可见性：图片列表显示待处理、已完成、不可用、失败或过大状态；识别完成后，预览底部显示 OCR 文字摘要，OCR 命中仍由本地 FTS 查询返回。
- 设置页自适应：按内容区真实宽度在单列和双列间切换，避免紧凑窗口拥挤；每个设置组按自身内容高度布局，不再出现同排强制等高造成的大面积留白。
- 来源应用图标恢复：修复原生摘要层主动丢弃图标数据的问题；只有无法安全获取图标时才回退为首字母。

## 隐私与资源边界

缩略图和 OCR 均在本机完成。缩略图 IPC 只接受数据库内的 PNG 图片记录，限制源图片长度、解码分配和像素边界，并只返回最长边 96 像素的 PNG data URL；不会上传图片。OCR 继续只使用 Windows 已安装语言包，不下载模型，不上传图片或识别文字。剪贴板正文、图片和 OCR 结果仍只保存在本机 SQLite 中。

## 验收状态

本轮增加了原生摘要图标、受限缩略图 IPC、前端严格响应校验、图片列表/预览 OCR 可见性、设置页自适应布局和键盘预览回归。完整命令、最终测试数量、安装包哈希与未完成证据记录在 [v0.7.0 验证报告](https://github.com/zkwi/QuickPaste/blob/main/docs/releases/v0.7.0-verification.md)。

真实机快捷键时延、10,000 次回贴、100,000 次捕获、多屏全 DPI、管理员窗口、Windows OCR 语言包、安装/覆盖安装/卸载和网络抽查继续保持 **pending real-machine**；自动化通过不能替代这些证据。

## 已知限制

- Windows OCR 的语言与质量取决于系统已安装语言包；不可用、过大或失败会显示对应状态，不会联网寻找替代服务。
- 缩略图只用于历史列表，预览和粘贴仍按需加载原始图片；超出既有图片安全边界的异常记录会拒绝解码。
- 安装包未做 Authenticode 签名，可能触发 SmartScreen；仅构建 NSIS，不提供 MSI。
- 自动回贴、管理员 helper、屏幕捕获排除和多显示器定位仍受目标应用及 Windows 行为影响，完整真实机器证据尚待补齐。
