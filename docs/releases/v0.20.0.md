# QuickPaste v0.20.0 发布说明

v0.20.0 将自动检查更新改为按本地日历日执行：启用自动检查时，每天第一次启动会在后台检查一次 GitHub Release，同日后续启动不重复请求。本版本以标准 GitHub Release 分发，不标记为 Pre-release；真实机长期矩阵仍保留为待验证项，因此不表述为已经全面验收。

## 主要变化

- 自动检查继续默认开启，并在启动后延迟 15 秒执行，避免与历史数据库、剪贴板监听和窗口初始化争抢资源。
- 跨本地午夜后，即使距离上次检查不足 24 小时也会进行当天检查；同一天无论检查成功、没有更新或网络失败，都不会在后续启动中自动重试。
- 用户当天手动检查过更新时，同样视为当天已经尝试；手动检查按钮本身不受每日限制，仍可随时再次使用。
- 后台网络失败继续保持静默；只有发现新版本时才显示非抢焦点通知，下载、SHA-256 校验和安装仍必须由用户主动确认。
- 设置页中英文文案明确说明“每天首次启动自动检查”。
- 项目版本治理改为偏积极策略：新增用户可感知能力、默认行为变化或成组体验改进默认提升 minor，不压成 patch。

## 数据、安全与兼容性

- 本版本不改变 SQLite schema、历史数据目录、剪贴板格式、管理员授权协议或设置 schema，无需迁移。
- 自动检查保留既有本地时间戳，并额外保存检查发生时的本地日期键；旧缓存会安全地多检查一次完成迁移，此后不会因切换时区而重新解释上次日期。不保存剪贴板内容、设置负载、搜索词、OCR 结果或设备标识。
- 已经关闭“自动检查”的用户会继续保持关闭；手动检查不受影响。
- 更新源仍固定为公开的 `zkwi/QuickPaste` GitHub Releases，不允许用户改写，也不保存 GitHub Token。
- 主程序继续保持普通用户权限；安装包仍只提供当前用户范围的 Windows x64 NSIS，不生成 MSI、MSIX 或 APPX。
- 当前发布二进制未进行 Authenticode 签名。
