# QuickPaste v0.6.0 发布说明

v0.6.0 把 QuickPaste 从小规模文本/图片历史升级为支持完整 Windows 剪贴板格式、大历史检索、整理复用和本地 OCR 的个人桌面工具。本版本以[未签名 GitHub Pre-release](https://github.com/zkwi/QuickPaste/releases/tag/v0.6.0)分发；真实机长循环仍保留为待验证项，因此不表述为稳定版已全面验收。

## 主要变化

- 文件与多文件：捕获和回贴 Windows `CF_HDROP`，提供打开与定位；数据库只保存路径和元数据，不复制原文件。
- 富文本：同一条记录保存 plain text、HTML 与 RTF，支持保留格式或纯文本粘贴，并明确记录未能保存的格式。
- 类型化动作：文本、代码、链接、图片和文件在快速面板与管理页使用同一动作规则。
- 大历史：版本化 SQLite、增量 CRUD、FTS5 中文子串/拼音/OCR/文件检索、组合筛选和 keyset 分页。
- 数据安全：按数量、图片字节和期限管理容量，显示真实磁盘占用；支持未加密 SQLite 备份、原子恢复、压缩和损坏隔离。
- 整理复用：单层集合、可编辑永久文本/代码片段，以及安全的跨页批量固定、移动和删除。
- 本地 OCR：默认开启、可关闭，只处理已进入历史的图片并使用 Windows 已安装语言；不下载模型、不上传图片或文字。
- 代码预览：仅在打开代码预览时加载 `highlight.js` core 和命中的有限语言模块，失败或超限回退纯文本。

## 隐私与数据保管

剪贴板正文、文件路径、图片哈希和 OCR 结果始终留在本机，不进入更新检查或其他网络请求。数据库及导出的备份没有 QuickPaste 应用层加密；备份可能包含完整正文、OCR 文字和本机路径，需由用户自行选择安全位置并管理副本。

普通运行没有远程遥测，也不生成本地验收指标。维护者只有通过仓库提供的隔离验收启动器显式启用后，才会在临时 profile 中写入固定白名单的计数与耗时；其中不含正文、路径、查询词、哈希或设备标识。

## 已知限制

- Windows OCR 的语言与质量取决于系统已安装语言包；不可用、过大或失败会显示为对应状态，不会联网寻找替代服务。
- 单条 plain text、HTML 或 RTF 格式最多 8 MiB；单张图片源及持久化图片最多 64 MiB、单边最多 8192 像素且不超过 4000 万像素。超限附加格式会被省略；没有可安全保留正文的超限内容不会进入历史。
- 路径记录不会跟随、复制或修复原文件；文件被移动或删除后相关动作会禁用。
- 安装包未做 Authenticode 签名，可能触发 SmartScreen；仅构建 NSIS，不提供 MSI。
- 自动回贴、管理员 helper、屏幕捕获排除和多显示器定位仍受目标应用及 Windows 行为影响，完整真实机器证据尚待补齐。
