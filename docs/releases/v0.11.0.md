# QuickPaste v0.11.0 发布说明

v0.11.0 集中修复 v0.10.0 暴露的图片预览和中文 OCR 回归，并继续整理快速面板与管理页的对齐关系。本版本以标准 [GitHub Release](https://github.com/zkwi/QuickPaste/releases/tag/v0.11.0) 分发；真实机长循环仍保留为待验证项，因此不表述为已经全面验收。

## 重点变化

- 修复图片预览缩成细条的问题。图片现在使用全宽主预览，识别文字收在下方 72px 内的摘要区；640×440 紧凑窗口会进一步自适应，图片和 OCR 均保持可见。
- Windows 本地 OCR 不再盲目跟随用户配置语言，而是先枚举系统已安装识别器，按 `zh-Hans-CN`、`zh-CN`、`zh-Hans`、`en-US`、`en` 选择；没有匹配时才回退到用户语言引擎。
- 中文结果会移除识别器插入的汉字间空格。数据库升级到 schema v10 时，已有终态 OCR 结果会清除旧搜索投影并重新排队；从未启用 OCR 的图片保持不变。
- 快速列表来源轨道从 104px 收紧到 92px；管理页顶部工具栏改为稳定网格，来源、时间、OCR 状态和操作按钮使用一致轨道。

## 隐私与兼容性

- OCR 仍只使用 Windows 本机能力，不下载模型、不上传图片或识别文字。
- 数据库迁移在事务内完成；若迁移失败，原 schema 和历史内容保持不变。
- 安装包仍只提供当前用户范围的 Windows x64 NSIS，不生成 MSI。
- 从未开启本地 OCR 的图片不会因升级被自动加入识别队列。

## 尚待真实机验证

- Windows 10/11 的覆盖安装、卸载与重装。
- 100%–250% DPI、多显示器负坐标和任务栏边界。
- 中文图片批量重识别、长时间剪贴板监听、管理员目标回贴和 WebView2 缺失路径。

以上项目仍为 `pending real-machine`。维护者选择将 v0.11.0 作为标准 GitHub Release 分发，但这不改变上述验证状态。
