# QuickPaste v0.20.2 验证报告

验证日期：2026-07-23

本报告记录 v0.20.2 富文本格式包剪贴板序列修复的本机诊断、回归测试、人工复测、自动化和候选发布证据。本次不使用 Computer Use 或合成鼠标键盘输入；长期运行、完整安装生命周期和完整 DPI 矩阵继续单独列为 `pending real-machine`。

## 根因与缺陷回归

- 本机剪贴板数据库元数据与内容无关日志确认，近期成功的短文本记录只携带纯文本；较长、多行记录通常来自 Chromium/Codex 并同时携带 HTML，因此进入 `paste_clipboard_formats` 路径。
- 富文本失败时，格式包写入后立即观察到剪贴板序列发生二次变化，自动粘贴在初始剪贴板安全校验处停止；同一环境中的纯文本路径继续成功发送完整 `Ctrl+V` 输入序列。
- 根因是 `write_format_package` 在 `clipboard_win::Clipboard` guard 仍持有剪贴板时读取序列号，函数返回时才隐式关闭剪贴板，导致返回值早于最终稳定序列。
- 真实 Windows 剪贴板测试新增“写入返回序列必须等于函数返回后当前序列”的断言；修复前稳定出现 RED，显式关闭 guard 后再读取序列则转为 GREEN。
- 用户在本机安装候选修复后，使用 Codex 长文本及含换行文本完成真实交互复测并确认可用。
