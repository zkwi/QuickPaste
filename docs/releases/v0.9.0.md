# QuickPaste v0.9.0 发布说明

v0.9.0 改进了从快速面板回到原输入光标的可靠性，并把历史列表改为更高效的单行信息布局。本版本以 [GitHub Pre-release](https://github.com/zkwi/QuickPaste/releases/tag/v0.9.0) 分发；真实机长循环仍保留为待验证项，因此不表述为稳定版已全面验收。

## 主要变化

- 直接回贴：唤起时记住目标顶层窗口与实际输入焦点子窗口；双击、Enter 或 `Alt + 数字` 会恢复最小化目标、重试前台激活，并把焦点还给原输入位置后发送粘贴。
- 高密度列表：每条记录只展示一行有效内容，删除高重复率的标题/正文双行；来源与时间合并到同一轨道，标准窗口至少展示 10 条记录。
- 搜索上下文：正文默认优先；标题单独命中、拼音命中和长文本尾部命中仍展示真正相关的原文片段。
- 格式化内容：纯文本使用 Unicode，富文本把 plain text、HTML 与 RTF 一次写回，文件继续使用 `CF_HDROP`，图片保留现有原生兼容链路。
- 复杂对象降级：Office/OLE 等应用注册格式不盲目序列化进程相关句柄；可安全读取的 Unicode、HTML、RTF 或文件列表继续保存，不透明对象格式会明确标为已省略。
- 权限边界不变：普通主程序不提权；管理员目标仍只使用带时限、进程互认且绑定剪贴板序列的一次性 UAC helper，并携带同一焦点子窗口身份。

## 隐私与资源边界

本版本没有新增网络服务或运行时依赖。剪贴板正文、文件路径、图片和格式遗漏信息仍只保存在本机 SQLite，不进入更新检查。复杂对象不会把任意 OLE 句柄、延迟渲染流或对象引用持久化；超过既有大小边界的标准格式继续终止性省略。

## 验收状态

本轮自动化覆盖焦点子窗口身份、普通/管理员目标协议、复杂对象回退、单行搜索结果、双击类型化粘贴、键盘与中文输入法组合态。完整命令、测试数量、候选安装包摘要与未完成证据记录在 [v0.9.0 验证报告](https://github.com/zkwi/QuickPaste/blob/main/docs/releases/v0.9.0-verification.md)。

真实机快捷键时延、10,000 次回贴、100,000 次捕获、多屏全 DPI、管理员窗口、Office 复杂对象矩阵、安装/覆盖安装/卸载和网络抽查继续保持 **pending real-machine**；自动化通过不能替代这些证据。

## 已知限制

- 自定义 OLE/应用对象只有在能拆出受支持的标准格式时才进入历史；依赖源进程、COM 流或延迟渲染的私有格式不会原样重放。
- 自动回贴仍受 Windows 前台窗口策略和目标应用行为影响；目标身份或焦点复核失败时只保留可手动粘贴的副本。
- 仅构建 Windows x64 当前用户 NSIS，不提供 MSI；完整真实机器安装证据尚待补齐。
