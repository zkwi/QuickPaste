# QuickPaste v0.16.0 发布说明

v0.16.0 改进中文短查询、图片列表加载和更新失败后的恢复体验，并让高频使用后的排序、资料库筛选与系统“减少动态效果”偏好更符合直觉。本版本以标准 GitHub Release 分发，不标记为 Pre-release；真实机长循环仍保留为待验证项，因此不表述为已经全面验收。

## 主要变化

- 中文单字/双字搜索使用有界短词索引，输入增加轻量防抖；在 10,000 条合成历史基准中，短查询 p95 为 20.272ms，低于 50ms 门槛。
- 图片写入时预生成受限缩略图，旧图片首次读取后自动懒回填，历史列表不再反复解码完整 PNG。
- 双击成功使用一条记录后，该记录会成为最新记录并移动到列表首位。
- 从资料库进入快捷面板或设置页并返回时，会话内保留资料库搜索词（包括 `@来源`）、类型、收藏夹以及最近的全部/固定/图片分区；快捷面板自身仍按原逻辑重置，这些资料库筛选不会写入磁盘。
- 更新检查或下载遇到网络不可达、DNS/连接失败或超时时，可直接重试或打开官方 Releases 页面；待执行的自动检查不会覆盖手动失败状态。
- Windows 启用“减少动态效果”时，过渡和动画会立即完成。

## 数据与兼容性

- 本版本把历史数据库 schema 升级到 v12：新增图片缩略图缓存与短词搜索索引。升级在事务中完成，不改变剪贴板正文、集合、固定或永久片段数据；无需手动迁移。
- 数据库、WAL 和 SHM 继续保存在 `QuickPaste.exe` 同级的 `data` 文件夹。复制或迁移前必须从托盘完全退出 QuickPaste，避免遗漏仍在 WAL 中的数据。
- 安装包仍只提供当前用户范围的 Windows x64 NSIS，不生成 MSI。

## 尚待真实机验证

- 100%–250% DPI、多显示器负坐标和不同任务栏位置。
- 大历史库首次升级耗时、长时间剪贴板监听、管理员目标回贴和 WebView2 缺失路径。

以上项目保持为 `pending real-machine`，不作为已完成验证陈述。
