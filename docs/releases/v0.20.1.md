# QuickPaste v0.20.1 发布说明

v0.20.1 修复 Codex 等多进程桌面应用中自动粘贴可能间歇性退化为仅复制的问题。本版本以标准 GitHub Release 分发，不标记为 Pre-release；真实机长期矩阵仍保留为待验证项，因此不表述为已经全面验收。

## 主要变化

- 本机调试确认 Codex 的可见顶层窗口与实际输入渲染子窗口可能属于不同的 `ChatGPT.exe` 进程；旧实现只接受与顶层窗口同进程的焦点，因此会随渲染焦点状态变化而间歇性失败。
- 当启动快速面板时没有捕获到明确焦点子窗口，恢复目标后允许当前焦点位于原目标根窗口树中的协作渲染进程，从而继续发送 `Ctrl+V`。
- 已捕获到明确焦点子窗口时仍要求精确句柄与进程匹配；前台根窗口、剪贴板版本、物理修饰键、helper 授权时限和管理员权限边界保持不变。
- 修复前新增的跨进程焦点回归测试按预期失败；最小修改后与既有精确焦点测试共同通过。

## 数据、安全与兼容性

- 本版本不改变 SQLite schema、历史数据目录、剪贴板格式、设置 schema 或管理员授权协议，无需迁移。
- 放宽只发生在无法捕获精确焦点子窗口且实际焦点仍属于原目标根窗口的场景；不会接受其他顶层窗口或其他应用的焦点。
- 主程序继续保持普通用户权限；管理员窗口仍只通过现有一次性 UAC helper 粘贴。
- 安装包仍只提供当前用户范围的 Windows x64 NSIS，不生成 MSI、MSIX 或 APPX。
- 当前发布二进制未进行 Authenticode 签名。

## 尚待真实机验证

- Chrome/WebView2、Office、记事本、远程剪贴板工具和管理员窗口组合下的长期焦点兼容。
- Windows 10/11 覆盖安装、卸载、保留 `data` 与重装。
- 500 次暖唤起、10,000 次外部普通目标粘贴、100,000 次独立捕获与完整 DPI 矩阵。

以上项目保持为 `pending real-machine`，不作为已完成验证陈述。
