# QuickPaste v0.21.0 发布说明

v0.21.0 继续收紧 Windows 剪贴板资源释放与稳定序列观察之间的契约，并把仓库首页重构为更易使用的中英文入口。本版本以标准 GitHub Release 分发，不标记为 Pre-release；真实机长期矩阵仍保留为待验证项，因此不表述为已经全面验收。

## 主要变化

- 敏感剪贴板历史标记触发提前跳过时，QuickPaste 现在先关闭显式 Windows 剪贴板 guard，再读取用于去重的终态序列。这消除了与 v0.20.2 富文本写入问题同类的生命周期时序窗口。
- 新增独立析构顺序回归，明确要求 guard 先释放、序列后观察；真实 Windows Chrome 风格多段 HTML 往返、内部写入抑制、文本/图片回读和管理员 helper 的精确序列校验继续保留。
- `README.md` 改为以快速开始、核心能力、快捷键、安装数据、隐私安全和开发流程为主线，减少过长的平铺能力清单。
- 新增语义等价的 `README.en.md`；版本治理会持续检查两份 README 的互链和当前 Release URL，避免文档版本漂移。

## 审计结论

- 显式 `clipboard-win` guard 只出现在 Windows 格式包读取与写入路径；普通读取、敏感内容跳过、文件写入和 HTML/RTF 写入现在都在终态序列观察前释放 guard。
- `arboard` 的文本与图片读取/写入在单次方法返回前结束系统剪贴板访问，外层 Rust 对象不表示持续持有 `OpenClipboard`，因此不需要复制显式 guard 的处理。
- 直接粘贴和一次性管理员 helper 仍只接受非零且完全匹配的稳定序列；审计没有增加忽略序列变化后继续粘贴的降级路径。

## 数据、安全与兼容性

- 本版本不改变 SQLite schema、历史数据目录、剪贴板载荷格式、设置 schema 或管理员授权协议，无需迁移。
- 目标窗口、焦点、剪贴板版本、物理修饰键、helper 授权时限和管理员权限校验保持不变。
- 主程序继续保持普通用户权限；管理员窗口仍只通过现有一次性 UAC helper 粘贴。
- 安装包仍只提供当前用户范围的 Windows x64 NSIS，不生成 MSI、MSIX 或 APPX。
- 当前发布二进制未进行 Authenticode 签名。
