# QuickPaste v0.21.1 验证报告

验证日期：2026-07-24

本报告记录 v0.21.1 间歇性闪退排查、焦点与输入法修复、自动化检查、Windows 构建和 GitHub 发布证据。长期运行、完整 DPI 与外部应用组合矩阵继续单独列为 `pending real-machine`。

## 根因证据与修复边界

- Windows Application 事件在同一天记录两次 `quickpaste.exe` 于 `ntdll.dll` 中以 `0xc0000374` 退出；这是原生堆损坏，不是无选中项导致的前端异常。
- WER 模块清单显示其中一次崩溃加载了微调输入法组件。现场同时存在系统剪贴板被微调用户进程持续独占的状态；重启这些用户进程后 `OpenClipboard` 立即恢复，完整 Rust 剪贴板测试随后通过。
- 快速面板唤起原先会调用 `AccessibleObjectFromWindow(OBJID_CARET)` 并接管 `IAccessible` COM 接口。v0.21.1 删除该进程内 MSAA 路径与对应 Windows feature，只保留 `GetGUIThreadInfo` 和附加线程 `GetCaretPos` 回退。
- 粘贴编排在进入会隐藏窗口的原生调用前主动结束输入法组合态并释放当前 WebView 焦点；仅复制或失败时恢复原焦点，成功粘贴仍保持原有隐藏行为。
- 新增回归测试禁止重新引入进程内 MSAA 调用，并覆盖无选中项 `Enter`、粘贴前失焦和失败后焦点恢复。
