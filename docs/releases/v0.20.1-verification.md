# QuickPaste v0.20.1 验证报告

验证日期：2026-07-23

本报告记录 v0.20.1 Codex 多进程焦点兼容修复的本机诊断、回归测试、自动化和候选发布证据。本次不使用 Computer Use 或合成鼠标键盘输入；Codex 完整交互、长期运行、安装生命周期和完整 DPI 矩阵继续单独列为 `pending real-machine`。

## 根因与缺陷回归

- 本机 Win32 只读探针确认 Codex 可见顶层窗口为 `Chrome_WidgetWin_1`，属于一个 `ChatGPT.exe` 进程；其 `Intermediate D3D Window` / `Chrome_RenderWidgetHostHWND` 输入后代属于另一个 `ChatGPT.exe` 进程，但 `GetAncestor(GA_ROOT)` 仍严格指向原 Codex 顶层窗口。
- 旧实现捕获焦点时拒绝跨进程后代并保存为“没有明确子窗口”；恢复目标后若 Windows 报告实际焦点为协作渲染进程，严格的同 PID/顶层句柄判断会返回失败。渲染焦点动态变化解释了同类文本有时成功、有时退化为仅复制。
- 回归测试先描述“没有捕获明确子窗口时，焦点位于不同进程但根窗口仍为原目标”的场景；旧实现按预期出现 RED。
- GREEN 阶段只调整无明确子窗口的验证分支：要求非空焦点且根窗口精确匹配原目标。已有明确子窗口时仍要求精确句柄与原目标进程匹配。
- 三项焦点快照定向测试共同通过，覆盖跨进程协作渲染窗口、无子窗口时的目标自身，以及明确子窗口的句柄/根窗口/进程拒绝边界。

## 自动化与安全审计

- `npm ci`：成功，从锁文件安装 140 个包。
- `npm run check`：退出码为 0：
  - 治理测试 46 项通过，版本、仓库、隐私、运行时边界和第三方声明检查全部通过；
  - 前端 34 个测试文件、730 项测试全部通过，TypeScript 类型检查与 Vite 生产构建通过；
  - Rust 格式检查、Clippy `-D warnings` 与 322 项测试全部通过。
- 真实剪贴板往返测试会与正在运行的 QuickPaste 监听竞争；精确停止已安装进程后，标准 `npm run check` 完整门禁通过，测试结束后从原路径恢复运行，历史数据库保持存在。
- `npm audit --registry=https://registry.npmjs.org --omit=dev --audit-level=high`：0 个漏洞；默认镜像不提供审计端点，因此显式使用 npm 官方 registry 完成审计。
- `cargo audit --file src-tauri/Cargo.lock`：退出码为 0，无阻断漏洞；报告 17 条已允许的未维护或 unsound 上游警告，来自 Tauri 跨平台依赖树。
- `npm run build:windows`：退出码为 0，只生成 v0.20.1 x64 NSIS 候选安装包，未生成 MSI、MSIX 或 APPX。
