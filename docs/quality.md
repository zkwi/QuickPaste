# 持续质量改进

QuickPaste 采用轻量、可重复的缺陷闭环。目标不是用流程替代判断，而是让已经出现过的问题不会再次悄悄进入候选版本。

## 缺陷闭环

1. 记录可复现步骤、Windows build、DPI、显示器布局、安装方式和脱敏证据。
2. 按用户影响分级：P0 为数据破坏或权限边界失守，P1 为主流程不可用或静默丢数据，P2 为明确的稳定性或高频体验缺陷，P3 为一般完善项。
3. 行为缺陷先增加失败的最小回归测试；原生窗口、权限和安装器问题同时更新 Windows 人工矩阵。
4. 实施最小修复，运行定向测试，再运行 `npm run check`。原生窗口或安装配置变化还要运行 `npm run build:windows`。
5. 用户可感知变化写入 `CHANGELOG.md`；尚未解决的风险保留在问题列表，不以模糊文案掩盖。

## 自动化防线

- `npm run check:versions` 同时检查版本、NSIS 契约、工具链和自定义标题栏所需的 Tauri 权限，防止前端 API 与能力清单漂移。
- 前端测试覆盖焦点、中文输入法组合态、搜索、删除撤销、异步乱序和窗口操作单飞。
- Rust 测试覆盖剪贴板读取退避、去重提交时机、SQLite 配置、窗口定位和提权协议边界。
- Windows CI 对每次主分支推送和拉取请求运行完整 `npm run check`。

## 候选版本复核

候选版本不得存在未说明的 P0/P1。每轮候选至少复核：

- 自定义标题栏按钮、系统标题栏双击和 Snap 后的最大化/还原。
- 冷启动、历史重载、剪贴板短暂占用、暂停恢复和敏感应用切换下的数据完整性。
- 紧凑/最大化窗口、浅色/深色、高对比度、中文输入法、100%–150% DPI 和双显示器。
- 普通窗口与管理员窗口粘贴、UAC 取消、目标变化和失败降级。
- NSIS 当前用户安装、覆盖安装、卸载和重装。

发布版原生诊断仅记录错误类型和操作上下文，不记录剪贴板正文；日志按大小轮转。提交问题前必须移除用户名、路径、聊天内容和其他隐私数据。
