# QuickPaste v0.3–v0.6 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 交付 QuickPaste `0.6.0`，完整实现已批准的 v0.3–v0.6 路线图和可重复验收工具。

**Architecture:** 保持 Windows/Tauri 模块化单体，把 SQLite 改为类型化、版本化、增量和可分页的唯一真值；Windows 格式包/OCR/系统动作放在独立 Rust 模块，Vue 只编排 domain/platform 与管理 UI。快速面板不承载集合编辑、批量、备份或容量管理。

**Tech Stack:** Vue 3、TypeScript 6、Vitest、Tauri 2、Rust 1.88、rusqlite bundled SQLite/FTS5、clipboard-win、Windows WinRT OCR、Playwright、highlight.js core。

## Global Constraints

- Windows 是唯一交付平台；主程序保持普通权限，管理员粘贴只走现有一次性 helper。
- 文件记录只保存路径和元数据，不复制源文件。
- 剪贴板正文不得进入网络；只允许现有更新器访问固定 GitHub 端点。
- 核心安装包不得捆绑 OCR/翻译模型、FFmpeg 或 ffprobe；只构建 NSIS。
- 快速面板只承担搜索、选择、预览和粘贴；管理操作进入管理页。
- 暖启动快捷键到首帧 p95 ≤120ms；10,000 条历史搜索 p95 ≤50ms。
- 普通窗口自动回贴成功率 ≥99.5%；漏记、异常重复和自身回流合计 <0.1%。
- 100%–250% DPI 和多显示器下无裁切、错位。
- 不提交、推送、打标签或发布；不等待 GitHub CI。
