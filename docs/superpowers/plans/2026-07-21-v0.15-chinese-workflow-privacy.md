# QuickPaste v0.15.0 中文工作流与隐私增强实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 发布 QuickPaste `0.15.0`，补齐 Windows 敏感剪贴板尊重、OCR/二维码直接使用、`;` 永久片段调用、`@来源` 建议、固定语义与快捷键提示。

**Architecture:** 保持现有 Windows/Tauri 模块化单体。隐私标记在原生剪贴板入口拦截；快速搜索语法先由纯 TypeScript 解析，再映射到 SQLite 类型化查询；OCR 复用现有文字复制/回贴；二维码只在图片预览时从 SQLite 读取 PNG 并在本机按需解码，不落库、不联网。

**Tech Stack:** Vue 3、TypeScript 6、Vitest、Tauri 2、Rust 1.88、SQLite/FTS5、clipboard-win、Windows OCR、`rqrr 0.10.1`、Playwright。

## 全局约束

- Windows 是唯一交付平台；主程序保持普通权限，管理员粘贴继续走现有一次性 helper。
- `ExcludeClipboardContentFromMonitorProcessing` 存在，或 `CanIncludeInClipboardHistory` 的序列化 DWORD 为 `0` 时，QuickPaste 不读取和保存该次内容。
- `CanUploadToCloudClipboard=0` 只表达禁止云同步；QuickPaste 是本地历史，因此不据此删除本地记录。
- `;` 和全角 `；` 仅在搜索串首字符出现时进入永久片段模式；`@` 建议支持鼠标、方向键、Enter、Escape，并尊重中文输入法组合态。
- 二维码正文只在当前预览会话内存中存在；只允许通过现有安全 URL 边界打开 `http/https` 链接。
- 不提交数据库、日志、截图、构建目录或任何真实剪贴板内容。
- 版本按功能提升为 minor：`0.14.1` → `0.15.0`。
