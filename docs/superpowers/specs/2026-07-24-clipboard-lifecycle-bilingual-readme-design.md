# QuickPaste v0.21 剪贴板生命周期审计与双语 README 设计

日期：2026-07-24

## 背景

v0.20.2 修复了富文本格式包在 Windows 剪贴板仍打开时提前读取序列号的问题。该缺陷只在 HTML/RTF 等格式包路径暴露，说明单纯按数据类型测试不足以覆盖资源释放与稳定状态观察之间的时序契约。与此同时，当前中文 README 的能力清单过长，安装、快速使用、数据位置和开发入口不够突出，仓库也缺少对应的英文首页。

本版本按项目积极版本策略定为 v0.21.0：它同时建立剪贴板生命周期回归边界、处理审计发现的同类风险，并提供结构化的中英文项目入口。

## 目标

1. 审计文本、HTML/RTF、图片、文件、敏感内容跳过、内部写入抑制和管理员 helper 粘贴路径中的剪贴板生命周期。
2. 对所有“读取内容或写入内容后再观察序列号”的路径建立明确契约：系统剪贴板必须已经关闭，观察值才可作为稳定状态参与去重、验证或粘贴授权。
3. 只修复有代码证据或失败测试证明的同类问题，不重写现有剪贴板抽象。
4. 重构中文 README 的信息层级，并新增语义等价的 `README.en.md`。
5. 让治理测试持续检查双语 README 的互链、当前版本和 GitHub Release 链接。

## 非目标

- 不统一重写 `arboard` 与 `clipboard-win`，不引入新的剪贴板框架。
- 不放宽目标窗口、焦点、剪贴板版本、修饰键、UAC helper 或管理员权限校验。
- 不改变 SQLite schema、历史数据目录、设置 schema 或剪贴板载荷限制。
- 不新增截图、云同步、远程遥测、云 OCR 或跨平台实现。

## 剪贴板生命周期设计

审计以资源生命周期而不是数据长度为入口：

- `clipboard-win` guard 代表一次显式 `OpenClipboard` / `CloseClipboard` 生命周期。任何用于稳定性判断的序列号必须在 guard 被释放后读取。
- `arboard` 的 `get_*` / `set_*` 调用在方法返回前完成系统剪贴板访问；其外层 `Clipboard` 对象不等同于持续持有 Windows 剪贴板。
- 格式包读取的普通路径继续通过 `read_package_with_guard` 在释放 guard 后观察序列。敏感内容跳过等提前返回路径必须遵守同一顺序，不能在 guard 仍存活时构造终态。
- 格式包写入继续在所有请求格式完成后显式释放 guard，再返回最终序列。文本、图片写入则在 `arboard` 写入方法返回后进行独立内容回读和前后序列一致性校验。
- 直接粘贴与管理员 helper 继续只接受非零且完全匹配的稳定序列；审计不会增加“忽略变化后继续粘贴”的降级分支。

若审计发现某一结构看似相似但资源释放不会改变被观察状态，则记录为已检查，不为形式一致性扩大生产改动。

## 测试设计

实施遵循 RED–GREEN：

1. 为敏感内容跳过的提前返回路径提取最小可测试边界，测试 guard 的析构必须先于序列观察；旧实现应因顺序相反而失败。
2. 保留并运行真实 Windows Chrome 风格多段 HTML 往返测试，覆盖写入关闭后的稳定序列。
3. 运行现有文本、图片、文件、内部写入抑制、剪贴板竞争和粘贴安全测试，证明没有放宽校验。
4. 为版本治理增加双语 README 契约测试：
   - `README.md` 链接 `README.en.md`；
   - `README.en.md` 链接 `README.md`；
   - 两者展示与 `package.json` 相同的版本；
   - 两者指向对应的正式 GitHub Release URL。

真实 Windows 剪贴板测试运行时必须暂停已安装 QuickPaste，避免监听器与测试竞争；测试结束后恢复或安装新候选版本。

## README 信息架构

`README.md` 保持仓库默认中文首页，`README.en.md` 为英文入口。两者采用相同章节顺序和事实边界，但英文版按英文技术文档习惯表达，不做机械逐句翻译：

1. 语言切换、产品名称、短定位与当前 Release。
2. 快速开始：下载安装、启动、复制、`Ctrl + Shift + V` 搜索并粘贴。
3. 产品截图。
4. 核心能力：快速检索、原生格式、Windows 粘贴、本地数据、OCR/二维码、管理与更新。
5. 常用快捷键表。
6. 安装、绿色版数据位置、迁移与 WebView2 约束。
7. 隐私、安全和明确的产品边界。
8. 开发、测试、Windows 构建、项目结构和深入文档。
9. 许可证状态与第三方声明。

当前 README 中过细的实现说明不再堆在单一能力列表中；仍有用户决策价值的限制保留，底层细节改为链接 `docs/architecture.md`、`docs/testing.md`、`SECURITY.md`、`CHANGELOG.md` 和当前 Release 说明。

## 发布

- 版本号统一提升为 `0.21.0`，更新 CHANGELOG、README、Cargo/Tauri/npm 元数据、第三方锁文件摘要、发布说明和验证报告。
- 发布前执行 `npm ci`、`npm run check`、npm/Rust 依赖审计和 `npm run build:windows`。
- 只发布 Windows x64 当前用户 NSIS 与包含主程序和四份第三方声明的绿色版 ZIP。
- 先创建 Draft Release，认证回下载核验文件名、大小、SHA-256 与 GitHub digest；通过后发布为正式 Latest Release，再匿名回下载复核。
- 完整真实机长循环继续标记为 `pending real-machine`，不得表述为已经全面验收。
