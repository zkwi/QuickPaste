# QuickPaste v0.13 首次体验与托盘唤起设计

## 目标与边界

v0.13 让首次使用路径从静态介绍变成一次可跳过、可完成的快捷粘贴练习，同时统一标题栏和托盘交互。首次引导只在既有 `onboardingCompleted=false` 时出现；窗口在当前显示器工作区居中，而日常快捷键和托盘唤起仍优先跟随文本光标、失败时跟随鼠标。顶部非交互区域支持按住左键拖动，按钮、输入框等交互元素不触发拖动。

引导最后一步提供“添加示例并练习”和“不添加”两个明确选择。示例由应用直接写入本地历史数据库，使用 QuickPaste 自有稳定标识，不读取、不清空、也不写入 Windows 系统剪贴板。用户选择练习后，快速面板显示一条可关闭的紧凑提示：先切换到任意输入框，按当前全局快捷键唤起，再对示例按 Enter。快捷键唤起时优先选中示例；只有原生回贴成功后才自动结束练习，单纯复制到剪贴板不算完成。用户可随时关闭提示，示例记录仍可像普通历史一样删除。

## 实现与验证

前端窗口适配层增加 `center()` 和 `startDragging()` 的失败降级封装，并只在首次引导挂载时请求居中。设置结构增加一个向后兼容的练习待完成布尔值。示例复用现有历史持久化通道，避免新建数据库接口。Rust 端把快捷键内联逻辑提取为共享激活函数，托盘左键调用同一路径，从而共同完成前台目标捕获、caret/鼠标锚点选择、窗口定位和会话通知。

回归验证覆盖：窗口 API 调用与失败降级；首次引导居中；跳过不写历史；选择练习只产生历史 mutation、不调用任何剪贴板写入；练习提示、示例预选和成功回贴后收起；托盘与快捷键源码都调用共享激活入口。界面在标准与 640×440 紧凑窗口、浅色和深色主题中检查焦点、无溢出和中文排版。发布仍只提供 Windows x64 NSIS 与绿色版 ZIP，并作为正式 GitHub Release。
