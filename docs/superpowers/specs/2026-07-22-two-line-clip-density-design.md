# QuickPaste 两行条目信息密度设计

日期：2026-07-22

目标版本：v0.19.0

## 背景

快速面板当前把条目摘要限制为单行，较长文本很快被省略。管理页虽然占用两行，但普通文本和代码记录的标题来自正文首行，导致“标题 + 正文”经常展示近乎相同的信息。两处界面都没有充分利用已有垂直空间。

## 目标

- 快速面板默认展示最多两行有效摘要，同时保持标准模式 44px、紧凑模式 40px 的现有行高。
- 管理页把重复的标题与正文合并为一个最多两行的有效摘要，不增加现有行高。
- 保留搜索高亮、OCR/拼音/缺失文件状态、来源、时间、批量选择和行操作。
- 保持现有悬浮预览规则：只有两行摘要仍被截断时才延迟显示全文预览。

## 非目标

- 不修改剪贴板捕获、粘贴、数据库 schema、历史分页或搜索协议。
- 不增加用户设置或密度切换选项。
- 不压缩来源、时间、复选框或操作按钮的可点击区域。
- 不调整管理页整体导航、筛选器或批量工具栏。

## 展示规则

### 快速面板

所有条目继续使用现有 `quickClipText` 选择有效摘要，文本区域由单行省略改为两行截断：

- 文本和代码：标题只是正文前缀时直接显示正文；自定义标题与正文不同时显示“标题 · 正文”。
- 链接：显示 URL/正文。
- 单文件：显示文件内容摘要；多文件显示标题与文件摘要。
- 图片：显示安全标题，不展示 data URL。
- 搜索时若只有标题命中，则优先显示标题，保证命中词可见。

标准与紧凑模式继续使用 18px 行高；两行内容分别占 36px，在现有 44px/40px 条目高度内居中显示。图标、来源、时间和行操作保持原位置。

### 管理页

管理页移除固定的“粗体标题一行 + 灰色正文一行”组合，改为一个两行摘要区域。摘要选择规则与快速面板一致，但使用管理页搜索词和高亮器：

- 普通捕获的文本/代码优先展示更多正文，不重复首行标题。
- 永久片段或其他自定义标题与正文不同时，保留“标题 · 正文”。
- 图片仅展示图片标题、尺寸等安全摘要，并保留 OCR 状态。
- 文件与链接保留其有意义的标题或内容组合。
- 仅标题命中搜索时仍展示标题；OCR/拼音命中徽标继续显示。

管理页行高、图标、原生复选框、来源时间列和操作列保持不变。提升来自“两行都承载唯一信息”，不是缩小交互区域。

## 实现边界

- 在 `App.vue` 邻近现有 `quickClipText` 的位置增加管理页摘要选择逻辑，复用现有规范化、搜索预览与特殊类型规则，避免引入新依赖或大范围重构。
- `QuickPanel.vue` 只调整摘要 CSS 行为，不改变条目事件和粘贴路径。
- `LibraryManager.vue` 将标题和正文节点替换为单一摘要节点，状态徽标保持独立，避免徽标进入截断文本。
- CSS 使用两行 line clamp、正常换行和长词断行；保留 overflow，确保现有“是否确实截断”检测继续基于实际尺寸工作。

## 无障碍与交互

- 快速面板和管理页现有 `list` / `listitem`、组合框、复选框和键盘焦点模型不变。
- 行的可访问名称仍提供标题与来源；视觉摘要去重不删除底层剪贴板信息。
- 搜索高亮继续作用于实际展示的摘要，标题独占命中不会变成不可见结果。
- 双击粘贴、Enter、Alt+1…0、空格预览、批量选择和行操作不改变。

## 验证

- 先添加失败测试，覆盖快速面板两行样式、标准/紧凑行高不变、管理页不再渲染重复标题正文，以及不同标题、图片、文件、链接和标题独占搜索命中的摘要选择。
- 定向测试通过后运行完整 `npm run check`。
- 使用 Playwright CLI 检查 800×580 浅色标准窗口、640×440 深色紧凑窗口和管理页，确认两行截断、可见条目数、搜索高亮、徽标布局、键盘焦点与控制台错误。
- 运行 `npm run build:windows`，核验唯一 x64 NSIS 与绿色版产物、版本、PE 架构、文件数量和 SHA-256。

## 发布

该改动同时改变快速面板与管理页的主要信息布局，按项目版本规则提升 minor 版本至 v0.19.0。发布说明明确记录无数据库迁移、未完成的真实机矩阵以及未签名状态。
