# QuickPaste v0.10 界面精简设计

## 目标

本轮以“更高信息密度、更少重复控件、更直接的更新入口”为目标，保持现有 Windows/Tauri 架构、数据安全边界和主要操作能力不变。

## 快速面板与图片预览

- 快速列表继续使用单行正文，把来源应用和相对时间收进同一个纵向元信息单元，减少右侧占宽并给正文留出更多空间。
- 类型图标或图片缩略图继续紧邻正文；图片预览在宽窗口中并排展示图片和 OCR 摘要，窄窗口再自然回落为上下排列。
- 不删除影响格式完整性判断的警告，只压缩其视觉占用。

## 剪贴板管理页

- 移除顶部“历史条目 / 长期固定 / 本地”三张重复统计卡；总数与固定数量已经分别存在于结果状态和侧边栏。
- 管理搜索与类型筛选合并为同一工具栏；保留“全部、文本、代码、链接、图片、文件”，移除可由搜索和左侧“已固定”入口替代的精确来源、固定状态筛选。
- 来源应用与时间合并为一个纵向元信息单元。批量操作在未选择记录时只显示选择摘要，选中后再出现移动、固定和删除动作。

## 设置与更新

- 存储统计仅显示“数据库大小”和“数据条数”，大小统一换算为 MB；移除 WAL、SHM、逻辑载荷、时间范围和容量策略编辑器等面向普通用户价值较低的信息。
- 备份、恢复和安全压缩仍保留，因为它们属于数据维护能力而非统计项。
- 自动或手动检查发现新版本时，在右下角显示带“下载安装”按钮的操作通知；下载、校验、落盘历史和启动 NSIS 的现有安全顺序保持不变。

## 验证与发布

- 为筛选精简、两项核心统计、来源/时间合并、批量操作渐进展示和更新通知补充回归测试。
- 通过真实浏览器检查主页面、管理页、设置页、紧凑宽度和深浅色主题，并把主页面与设置页截图加入 README。
- 作为 `0.10.0` minor 版本发布；本地门禁和 Windows NSIS 构建通过后按现有规则标记为 Pre-release，真实机长循环继续保留 `pending real-machine`。
