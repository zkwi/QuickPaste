# QuickPaste v0.9 直接回贴与紧凑列表设计

## 目标

快速面板中的双击、Enter 和 `Alt + 数字` 应把所选内容直接送到唤起面板前的输入光标；列表应删除标题/正文重复形成的第二行，在现有窗口中显示更多历史。

## Ditto 参考与取舍

参考 Ditto 的 `ExternalWindowTracker`、`ProcessPaste` 与 `OleClipSource`：先保存外部窗口与焦点、把一组原生格式放回剪贴板，再恢复目标并发送粘贴按键。QuickPaste 采用相同顺序，但保留现有普通权限主进程、剪贴板序列复核、目标 PID 复核和一次性 UAC helper；不修改系统级 foreground lock timeout。

复杂对象按安全标准表示拆分。纯文本写 `CF_UNICODETEXT`；富文本原子写 plain text、HTML 与 RTF，由目标应用选择最优格式；文件写 `CF_HDROP`，不把路径伪装为文本；图片走现有 PNG/DIB 兼容链路。应用注册格式用于识别 Office/OLE 等复杂对象，但进程相关句柄、延迟渲染流和任意不透明对象不持久化，同一对象中的 Unicode/HTML/RTF/文件回退继续保存，并在记录中标明对象格式已省略。

## 交互与布局

唤起时保存顶层 `HWND`、PID 和 `GUITHREADINFO.hwndFocus`。粘贴时恢复最小化窗口，在有界时间内重试 `BringWindowToTop`/`SetForegroundWindow`，临时附加输入线程并把焦点还给原子窗口；任何身份、时限、修饰键或剪贴板序列复核失败都只保留可手动粘贴的剪贴板副本。

快速列表每项只显示一行内容。默认显示正文；只有标题单独命中或拼音命中时显示标题，使搜索上下文可见。类型图标、来源应用与相对时间保留，悬停操作覆盖元信息轨道。标准窗口目标至少 10 条、`640 × 440` 紧凑窗口至少 7 条。

## 验证

- Rust：焦点子窗口身份、管理员 helper 协议、复杂对象标准回退与遗漏标记。
- Vue/domain：双击类型化粘贴、标题/正文只显示一次、标题和拼音命中、浅色/深色、键盘与中文输入法组合态。
- 发布：`npm ci`、`npm run check`、Windows NSIS 构建、版本/隐私/产物核验，以及真实 Notepad 回贴和页面截图检查。
