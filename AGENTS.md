# QuickPaste 项目协作约定

## 项目边界

- Windows 是当前唯一交付平台；保留清晰的平台适配层，但不提前实现其他端。
- 优先保证高频交互、数据安全和可维护性，不为假设需求引入新框架或服务。
- 主程序必须保持普通用户权限；管理员窗口粘贴只能走现有的一次性授权 helper。

## 代码分层

- `src/domain/` 保持纯 TypeScript，不直接访问浏览器或 Tauri API。
- `src/platform/` 负责前端与系统能力之间的适配和失败降级。
- `src/App.vue` 负责界面编排；可复用规则应进入 domain 或 platform 层。
- `src-tauri/src/lib.rs` 承载 Windows、SQLite、剪贴板、窗口和托盘能力。

## 修改要求

- 不提交剪贴板数据库、日志、截图、构建产物、签名证书或任何真实用户内容。
- 搜索、删除、撤销、快捷键、焦点和窗口定位修改必须补充相应回归测试。
- UI 修改至少检查紧凑窗口、浅色/深色主题、键盘操作和中文输入法组合态。
- 安装包只使用 NSIS；发布配置不得重新引入 MSI。

## 版本定级

- 项目采用偏积极的版本策略：新增或扩大用户可感知能力、改变默认行为、显著调整 UI/交互，或形成一组可感知体验改进时，默认提升 minor，不因代码量小或兼容旧数据而保守降为 patch。
- patch 只用于范围明确、不会改变主要流程或布局结构的修复。
- 同一版本包含多项用户可感知变化或任何默认行为变化时，不得打包成 patch；按影响最高的变更定级。
- `1.0.0` 前的新能力或阶段性里程碑可以主动提高 minor；达到稳定交付标准时应考虑提升到 `1.0.0`，不为维持低版本号而延后。破坏性变化至少提升 minor，并在发布说明中记录迁移影响。

## 完成门槛

```powershell
npm ci
npm run check
```

涉及安装或原生窗口行为时，再运行：

```powershell
npm run build:windows
```

- 本地 `npm run check`、Windows 构建、隐私扫描和发布产物核验是发布阻断门槛。
- GitHub Actions 只作为推送后的异步复核；发布任务记录运行链接和当时状态，但不阻塞等待远端 CI 完成。
- 异步 CI 后续失败时应进入缺陷修复闭环，不得静默忽略；必要时撤下或标记受影响的 Release。

提交与发布由维护者明确发起；不要自动提交、推送、打标签或发布。
