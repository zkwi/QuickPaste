# 更新日志

本项目按语义化版本管理。这里只记录用户可感知变化；内部治理、重构和文档调整不进入产品日志。

## Unreleased

## 0.16.0 - 2026-07-21

让中文短查询、图片浏览和更新失败后的恢复更直接，同时保持管理筛选与系统减少动态效果偏好。

### Changed

- 中文单字/双字查询改用有界短词索引，输入搜索增加 120ms 防抖；长文本仍安全回退，来源、类型和集合组合查询保持原语义。
- 图片记录在写入时预生成受限缩略图，并在旧记录首次读取后懒回填，列表不再反复解码完整 PNG。
- 双击成功使用一条记录后，该记录会成为最新记录并移动到列表首位。
- 从资料库进入设置或快捷面板并返回时，会话内保留资料库搜索词（包括 `@来源`）、类型、收藏夹和最近分区；快捷面板自身仍按原逻辑重置，显式清除仍会生效。
- 系统启用“减少动态效果”时，界面过渡和动画立即完成，不再保留延迟。

### Fixed

- 更新检查或下载安装遇到网络不可达、DNS/连接失败或超时时，显示本地化恢复提示，并提供安全重试与官方 Releases 页面入口。
- 手动检查更新会取消待执行的自动检查，避免自动任务覆盖刚出现的失败恢复状态。

## 0.15.0 - 2026-07-21

围绕中文用户的高频检索、截图复用和隐私习惯，补齐永久片段、来源快捷筛选、本地二维码识别和更清晰的固定语义。

### Added

- 在搜索框输入 `;` 或 `；` 可只查看永久片段，输入 `@` 可按已记录的微信、飞书、WPS、浏览器等来源应用快速筛选，并继续叠加正文关键词。
- 图片预览中的 Windows 本地 OCR 结果可直接复制或粘贴；截图内二维码会在本机识别，文本可复制，安全的 HTTP/HTTPS 链接可直接打开。
- 设置页会提示 `Ctrl + Shift + V` 与纯文本粘贴、`Ctrl + Alt + V` 与选择性粘贴的常见快捷键语义冲突。

### Changed

- 标题栏“面板常驻”改用面板图标和明确文案，与条目的“固定内容”图钉分开；高频按钮补充操作和快捷键 Tooltip。
- Windows 剪贴板声明不应进入历史时，QuickPaste 会在读取正文前跳过记录；仅禁止云剪贴板上传的内容仍可保留在本机历史。

### Fixed

- 恢复快速列表中的条目固定入口，并保留键盘焦点、持久化和管理页状态同步。

## 0.14.1 - 2026-07-20

修复无边框快速面板从顶部拖动时可能立即消失的问题。

### Fixed

- 原生窗口拖动使 WebView2 短暂失焦时，只有顶层 QuickPaste 窗口也确实失去前台状态才自动隐藏；正常点击其他应用时仍保持原有自动收起行为。

## 0.14.0 - 2026-07-20

把安装版和绿色版统一为可直接复制的数据目录，并进一步修复首次引导的主动显示、居中和窗口拖动体验。

### Added

- 历史数据库、WAL 和 SHM 统一保存在 `QuickPaste.exe` 同级的 `data` 文件夹，完全退出后复制整个文件夹即可迁移。
- 设置页存储管理增加数据位置说明和“打开数据目录”入口，不向前端暴露用户绝对路径。

### Changed

- 首次引导由原生窗口状态托管：首次运行主动居中并保持可见，托盘或快捷键再次唤起时仍显示引导；完成后恢复日常跟随光标定位。
- 快速面板、管理页和引导页顶部统一使用 Tauri 原生深层拖动区域，非交互区域可像普通标题栏一样移动窗口。
- `0.14.0` 不读取或迁移开发阶段旧版本的应用数据目录；安装版升级和卸载会保留程序目录中非空的 `data` 文件夹。

### Fixed

- 修复首次引导可能因窗口失焦自动隐藏、安装后看似没有启动的问题。
- 修复部分 WebView2 环境中手写指针拖动与按钮命中冲突、导致顶部区域无法稳定移动窗口的问题。

## 0.13.0 - 2026-07-20

让首次启动、窗口移动和托盘唤起更符合 Windows 使用习惯，并提供不改写系统剪贴板的可选快捷粘贴练习。

### Added

- 首次引导可选择向应用历史添加一条欢迎记录，并通过当前全局快捷键完成第一次粘贴；也可以直接跳过。
- 欢迎记录使用稳定 ID 直接写入 QuickPaste 本地历史，不读取、替换或写入用户当时的系统剪贴板内容。
- 引导结束后显示可关闭的紧凑练习提示；再次唤起快速面板时会优先选中欢迎记录，成功粘贴后自动结束提示。

### Changed

- 首次安装的引导窗口会在当前显示器居中，完成引导后的日常快捷唤起仍继续跟随文本光标或鼠标。
- 左键点击托盘图标和托盘“打开”现在与全局快捷键使用同一条前台目标捕获、定位和显隐路径。
- 标题栏非交互区域支持按住鼠标拖动，按钮区域仍保持原有操作且不会误触拖窗。

### Fixed

- 修复托盘点击通过普通显示路径打开窗口，导致粘贴目标被清空且不能靠近当前光标的问题。
- 修复无边框快速面板顶部空白区域在部分 WebView2 环境中无法稳定拖动的问题。

## 0.12.0 - 2026-07-20

继续精简快速面板，把图片内容和高频操作放回视觉中心，并消除管理页与设置页在实际内容宽度下的重叠。

### Changed

- 快速面板移除底部目标应用、键盘提示和管理入口栏；目标应用与管理入口收进标题栏，为历史列表和详情释放完整底部空间。
- 图片详情把标题合并进返回栏，不再显示图片对象格式省略提示；OCR 结果默认折叠、可按需展开，标准与紧凑窗口都优先展示主图。
- 管理页搜索与操作保留首行，类型筛选独立放在可横向滚动的第二行；筛选按钮不再压缩成逐字换行。
- 设置页改为稳定单列内容轨道，顶部常用卡片、快捷键、更新操作和存储维护区按可用宽度自然换行。

### Fixed

- 修复来源应用图标、名称和时间在快速列表与管理列表中基线不齐的问题。
- 修复侧栏占用宽度后，设置页仍按整个窗口宽度判断列数、导致文字和控件相互挤压的问题。
- 修复图片详情在 640×440 紧凑窗口中仍被标题、格式提示、OCR 正文和底栏过度占用的问题。

## 0.11.0 - 2026-07-20

修复上一版图片预览与中文 OCR 回归，并继续收紧快速面板和管理页的对齐关系。

### Changed

- Windows 本地 OCR 会先枚举系统识别语言，并按简体中文、英文的顺序选择可用引擎；中文结果会清理识别器插入的汉字间空格。
- 管理页搜索、类型筛选、结果状态和主操作使用稳定网格对齐；OCR 状态统一放在标题行，来源和操作轨道进一步收窄。
- 快速面板的来源应用与时间轨道从 104px 收紧到 92px，为内容释放更多宽度。

### Fixed

- 修复图片与 OCR 双栏预览的百分比高度形成循环计算、导致图片缩成细条的问题；图片恢复为全宽主预览，OCR 改为底部低高度摘要。
- 修复 640×440 紧凑窗口中 OCR 摘要被图片最小高度挤出可视区的问题。
- 修复跟随用户配置语言时可能用英文 Windows OCR 引擎识别中文图片、只得到少量乱码的问题；升级时会原子清除旧 OCR 搜索投影并重新排队识别已有图片。

## 0.10.0 - 2026-07-20

进一步收紧快速面板、管理页与设置页的信息层级，把高频内容和更新入口放到更容易操作的位置。

### Added

- 检测到新版本时在窗口右下角显示可操作通知，可直接开始“下载安装”，并继续沿用下载摘要校验与安装前历史落盘保护。

### Changed

- 快速面板把来源应用与时间纵向合并到固定宽度的信息轨道，为正文释放更多横向空间；图片预览把图片与 OCR 文字并排展示。
- 管理页移除顶部统计卡片、来源精确筛选和固定状态筛选，只保留搜索与内容类型；未选择内容时不再显示整排不可用的批量按钮。
- 存储统计精简为“数据库大小”和“数据条数”，磁盘占用统一按 MB 显示，不再罗列 WAL、SHM、逻辑载荷和保留策略明细。

### Fixed

- 管理列表的来源与时间改为上下排列，并统一收紧行高、图标和操作间距，减少宽窗口中的视觉跳跃和冗余留白。

## 0.9.0 - 2026-07-20

让快速面板真正成为高密度、可直接回贴到原输入位置的操作面板。

### Changed

- 快速列表改为单行主内容，去掉标题与正文的重复展示；来源应用与时间合并到同一元信息轨道，标准窗口可显示至少 10 条记录。
- 富文本和 Office 等复杂剪贴板对象优先保留可安全持久化的 Unicode、HTML、RTF 或文件列表表示；不透明 OLE/应用注册格式会明确标记为已省略。

### Fixed

- 唤起面板时同时记住目标顶层窗口和实际获得输入焦点的子窗口；双击、Enter 与 `Alt + 数字` 粘贴前会恢复最小化目标、重试前台激活并还原原输入焦点。
- 普通窗口与管理员窗口共用同一份焦点目标身份，UAC helper 不再只恢复顶层窗口后把内容留在剪贴板。
- 拼音或标题单独命中时，单行结果会展示真正命中的原文，不会为了收紧布局而隐藏匹配上下文。

## 0.8.0 - 2026-07-20

统一设置、管理和图片预览布局，让 OCR 检索与自动更新更直接。

### Changed

- 设置页顶部新增“管理剪贴板”和全局快捷键两个常用入口，并整体放大正文、说明和按钮字体，保持统一的密度与层级。
- 管理页搜索、结果状态、新建片段和批量操作重新分组；批量工具栏会按可用宽度换行，不再发生文字重叠。
- 图片预览改为单行紧凑元信息，移除重复的图片格式徽标，把首屏空间优先留给图片。
- “下载安装”一次点击即下载、校验并静默启动 NSIS 安装，完成后重新打开应用。

### Fixed

- 搜索框和 OCR 设置明确说明识别文字可检索，并保持快速面板与管理页的本地 OCR 命中标记。
- 主界面移除低频暂停按钮和新手引导中的暂停宣传，暂停/恢复仍可由托盘和既有键盘命令完成。
- 不再在备份区域显示数据库未加密提示，也不再在更新界面强调安装包签名状态。
- 管理页“新建片段”恢复为清晰的主操作样式，紧凑窗口和深色主题下保持一致。

## 0.7.0 - 2026-07-20

集中修复图片浏览与设置页回归，让本地 OCR 和来源应用信息重新可见。

### Changed

- 设置页根据实际内容宽度在单列和双列之间自适应，短卡片不再被同排内容强行拉高。
- 图片预览压缩标题和格式区域，并让图片优先占用首屏剩余空间；识别完成后会在预览底部显示 OCR 文字摘要。
- 图片历史列表从原生数据库按需获取有界缩略图，不再为了显示缩略图加载完整图片正文。

### Fixed

- 修复原生历史摘要丢弃来源应用图标、导致应用全部退化为首字母的问题。
- 修复图片历史列表只有通用图片占位符、看不到缩略图的问题。
- 修复 OCR 虽已默认启用并参与检索，但列表与预览缺少可见状态、容易误判为未生效的问题。
- 修复紧凑设置页仍保持双列、文字和控件拥挤，以及短设置组出现大面积空白的问题。

## 0.6.0 - 2026-07-20

从格式完整性到大历史、整理复用和本地 OCR 的一次完整能力升级。

### Added

- 支持 Windows `CF_HDROP` 文件/多文件捕获与回贴；只保存路径和元数据，并提供打开与定位动作。
- 同一条记录可保存 plain text、HTML 和 RTF，支持保留格式粘贴与纯文本粘贴。
- 按记录类型和所在界面生成一致的右键动作，包括图片另存为、链接打开和缺失文件安全禁用。
- SQLite FTS5 中文子串全文检索、拼音/首字母索引，以及类型、来源、集合和固定状态组合筛选。
- 未加密 SQLite 备份、原子恢复、数据库损坏隔离、真实磁盘占用显示与安全压缩。
- 单层集合、可编辑永久文本/代码片段，以及管理页跨页批量固定、移动与删除。
- 对历史图片默认开启 Windows 本地 OCR；只使用系统已安装语言，不下载模型、不上传正文。
- 代码预览按需加载有限的语法高亮语言模块，失败或超限时安全回退纯文本。
- 内容无关的本地验收工具，用于记录暖唤起、回贴与捕获完整性；普通运行默认不启用、不生成指标文件。

### Changed

- 历史存储改为版本化、类型化 SQLite schema 和增量 CRUD；列表使用元数据分页，大 BLOB 在预览或粘贴时按需读取。
- 普通历史同时按记录数量、图片逻辑字节数和保留期限裁剪；固定项与永久片段保留。
- 搜索与集合、固定状态、来源及类型共用同一查询语义；管理页批量目标可安全覆盖当前查询的跨页结果。
- 快速面板继续只承担搜索、选择、预览和粘贴，整理、备份、容量与 OCR 设置集中在管理页。

### Fixed

- 数据库写入失败、恢复失败或启动损坏不再以空状态覆盖既有历史；损坏文件会隔离并暴露恢复位置。
- 多格式写回、自身回流抑制、摘要与完整载荷边界使用严格运行时校验；文本/HTML/RTF 以 8 MiB、图片以 64 MiB/8192 像素/4000 万像素为硬边界，避免异常正文进入数据库或在读取时造成无界分配。
- OCR 任务按图片哈希去重并使用有界单工队列；迟到、重复或已关闭任务不能覆盖新状态。

## 0.2.0 - 2026-07-19

面向紧凑窗口、高 DPI 和首次使用流程的体验改进版本。

### Changed

- 首次启动引导会按有效窗口高度自动收紧标题栏、演示区、正文和底栏，在 `640 × 440` 等紧凑窗口中仍保持完整信息层级。
- 引导切换步骤时根据真实溢出状态安排焦点，并阻止按钮自动获焦导致页面跳动。
- 引导遮罩铺满当前窗口，最大化与普通窗口使用一致的边界和背景处理。

### Fixed

- 修复高 DPI 或较小有效视口下，“下一步”自动获焦把品牌标题和“跳过”按钮滚出可视区的问题。
- 正常支持的紧凑尺寸不再出现多余的引导滚动条；系统大字体、长翻译等极端溢出仍保留可滚动兜底。

## 0.1.0 - 2026-07-19

首个公开候选版本。当前 Windows 安装包尚未进行 Authenticode 签名，因此只作为 GitHub Pre-release 分发。

### Added

- Windows 文本与图片剪贴板历史、搜索、高亮、筛选、固定、删除和撤销。
- 新捕获中文内容的全拼与首字母索引，可直接按拼音搜索且保持原文高亮语义。
- 动态显示来源应用图标，并为无法获取图标的应用提供稳定回退。
- 靠近文本插入点或光标的紧凑快速面板，以及管理页、设置页和首次启动引导。
- SQLite 本地持久化、系统托盘、全局快捷键、敏感应用排除和尽力而为的屏幕捕获排除。
- 面向普通窗口及管理员窗口的安全粘贴流程。
- 简体中文和英文界面、浅色与深色主题。
- 当前用户范围的 NSIS x64 候选安装包。
- 固定 `zkwi/QuickPaste` 的 GitHub Release 更新检查、下载进度、SHA-256 完整性校验和用户确认后的 NSIS 更新。

### Changed

- 产品名称统一为 QuickPaste（闪电剪贴板），技术标识统一为 `quickpaste`，桌面 Bundle identifier 更新为 `com.quickpaste.desktop`。
- 快速面板收紧标题栏、搜索、筛选、列表和底栏节奏；标准窗口可完整展示至少 8 条双行记录，紧凑窗口至少展示 6 条，同时保留清晰的点击目标与键盘焦点。
- 来源应用区使用固定图标、名称和时间轨道，长短不一的应用名称不会再破坏右侧对齐。
- 设置页在宽屏使用双列紧凑布局，管理页压缩标题、统计与列表节奏，默认窗口展示更多记录。

### Fixed

- 屏幕捕获保护不再默认开启，避免 QuickPaste 在截图工具中显示为空白；旧版默认值会在升级时一次性迁移为关闭，主动需要隐私保护的用户可在设置中重新开启。
- 同内容再次复制会刷新来源和时间；QuickPaste 自己的复制与回贴不再被重新收录并污染历史。
- 图片剪贴板不再被强制缩放为 2560 像素，回贴时保留原始分辨率。
- 单修饰键全局快捷键不再允许注册，避免占用 `Ctrl+C`、`Ctrl+V` 等常用编辑操作。
- 拼音搜索命中增加明确的可见标识，管理页空状态按搜索、图片、固定和全部历史分别说明。

- 修复管理器在系统最大化后无法通过自定义按钮还原的问题；最大化状态、图标、辅助文案和铺满布局现在保持一致。
- 冷启动或历史重载期间的剪贴板变化会先缓冲并在历史就绪后合并，不再因订阅时序直接丢失。
- 剪贴板被其他应用短暂占用后会退避重试；暂停记录或敏感应用排除不再污染内容去重状态。
- 管理器搜索在中文输入法组合态下不再抢占方向键，直接粘贴也不会被额外修饰键误触发。
- 窗口、固定面板和快捷键异步操作改为单飞执行，相对时间在界面长时间打开时会自动刷新。
- SQLite 历史连接会等待短暂写锁，窗口模式切换也不再忽略退出最大化失败。
- 管理员窗口的一次性粘贴请求改为进程互认的本地命名管道，不再接受可由同一用户进程伪造的落盘授权文件或目标参数。
- 管理员粘贴请求绑定写入后已读回验证的剪贴板序列，并在注入前重复核对，避免 UAC 等待期间粘贴内容被替换。
- 主程序检测到自身以管理员权限启动时提示并退出，防止常驻历史与界面扩大权限边界。
- 管理员授权请求改为单飞执行；界面超时后取消迟到请求，未结束的 UAC 流程不会继续堆积。
- 损坏、重复或类型不正确的持久化历史不会再直接进入界面；旧记录缺少搜索关键词列表时会安全迁移。
- 浏览器存储空间不足或被禁用时，界面不再因设置写入失败而中断。
